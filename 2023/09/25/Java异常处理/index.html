Java异常处理

异常的分类

均继承自Throwable。

类型 描述 常见类
编译时异常 超出程序员控制的情况，例如试图打开不存在的文件，网络连接或数据库连接问题，通过强制程序员处理或声明这些异常，来确保这些问题被正确处理。编译器要求调用可以抛出编译时异常的方法和代码必须放在try-catch或在方法声明中使用后throws，否则无法编译通过 IOException,SQLException,FileNoeFoundException
运行时异常 编译器不强制捕获或声明，由编程错误导致，比如除0,或者访问null引用，意味着有代码要修复 NullPointerException,ArithmeticException,ArrayIndexOutOfBoundsException
错误 由Java虚拟机生成并抛出的异常，表示严重的问题，不是由应用程序正常操作引起的，通常指示有关java运行时环境的问题，比如说系统资源不足，类定义不兼容，程序不对其做处理 OutOfMemoryError，StackOverflowError，NoClassDefFoundError

throw，throws和Throwable

描述 用途
throw 关键字 显式地抛出异常
throws 关键字 用于声明方法可能会抛出异常，通知方法调用者可能需要进一步处理或抛出异常
Throwable 错误和异常的超类 用作异常和错误类的基类，在java中，只有Throwbale及其子类的实例可以被throw关键字抛出，被catch关键字捕获

finally

try中return和finally的执行顺序？
return代码执行时会将局部变量保存在栈帧的顶部，然后finally修改的还是原来的栈帧位置的局部变量，最终返回的信息还是栈帧顶部的变量。所以finally在return关键字之后执行，但是不改变栈帧顶部信息。
如何try和finally中都有return呢？
finally中的return返回的是finally中的局部变量的信息，即finally中的return会覆盖掉try中的return。

相似概念 描述
final 关键字，修饰的类不能被继承，修饰的方法不能被重写，修饰的变量变常量
finally 异常处理结构的一部分，与try-catch一起使用，表示最终执行的代码，常用来做资源的释放
finalize Object类中的方法，用于垃圾回收。当GC决定回收摸一个对象时，该对象的finalize方法会被调用，给予对象在销毁前执行某些操作的机会。不被推荐使用

自定义异常

Java中的异常都是Throwable，Exception，RuntimeException的子类，创建自定义异常，就是创建对应的子类。
